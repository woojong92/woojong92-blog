vercel을 이용한 서비스 배포를 알아보자.

### **vercel**

- 정적 웹사이트 및 애플리케이션을 빌드 & 배포하기 위한 클라우드 플랫폼
- 프론트/백엔드 개발자들이 코드를 더 쉽게 배포하고 호스팅 할 수 있게 도와줌
- Github, GitLab, Bitbucket 등과 연동하여 CI/CD를 간편하게 설정할 수 있음
- 사용성과 확장성이 좋음
- Next.js와 시너지가 극대화됨. 빠르고 효율적인 웹 애플리케이션 개발 및 배포 환경 구축

### Vercel의 주요 기능

- 웹 애플리케이션 호스팅
- 빠른 배포를
- CI/CD 통합
- 환경 변수 관리
- 도메인 관리
- 서버리스 함수 기능 제공
- 애널리틱스 도구 제공

### **Vercel로 배포하기**

- Vercel CLI로 배포 :

  - 프로젝트 디렉토리에서 Vercel CLI를 사용해서 수동 배포를
  - 명령어: Vercel

- Github/GitLab 연동 배포 (Github import):
  - Github, GitLab과 Vercel를 연동하여 저장소에 변경사항이 푸시될 때 마다 자동으로 배포
- 환경 변수 설정

### Github import (CI/CD란?)

- Vercel의 Github Import은 CI/CD 프로세스이 일부이다.
- Continuous Integration(CI)

  - 개발자나 팀 코드 변경을 일일이 검사하지 않고도 코드와 품질과 안정성을 유지하도록 도와주는 프로세스이다.
  - 코드 변경이 이루어질 때마다 자동으로 테스트, 빌드 및 코드 검사를 수행하고, 오류를 미리 감지한다.

- Continuous Deployment(CD)
  - CD는 CI의 확장으로 코드 변경이 CI 테스트를 통과하면 자동으로 프로덕션 환경에 배포되도록 하는 프로세스
  - 이를 통해 새로운 기능과 버그 수정 사항이 빠르게 사용자에게 제공된다.

1. Vercel 계정 생성
2. 프로젝트 생성: 대시보드에서 "New Project"나 "Import Project" 버튼을 클릭하여 새 프로젝트를 생성한다.
3. Github Import: "Import Git Repository"를 선택하고 GitHub 계정에 로그인하여 원하는 레포지토리를 import 한다.
4. 배포 구성: Vercel은 프로젝트를 자동으로 감지하고 배포 구성을 설정함(빌드 및 환경변수 추가 설정)
5. 배포: 프로젝트를 배포하려면 "Deploy"버튼을 클릭 (이후에는 설정한 브랜치에 푸시하면 자동 배포)
