웹사이트 성능과 사용자 경험을 향상시키기 위해 폰트 최적화는 중요한 과제입니다. 웹 폰트는 콘텐츠의 가독성과 디자인의 완성도를 높여주지만, 잘못된 폰트 설정은 로딩 속도를 저하시켜 사용자 이탈을 초래할 수 있습니다. 이 글에서는 폰트 최적화의 중요성과 기본 개념을 다룹니다.

## 1. 웹 폰트와 시스템 폰트

웹 폰트와 시스템 폰트의 차이를 이해하면, 어떤 상황에서 어떤 폰트를 사용하는 것이 적절한지 알 수 있습니다. 웹 폰트는 일관된 디자인을 제공하지만 로딩 속도에 영향을 미칠 수 있습니다. 반면, 시스템 폰트는 빠르지만 디자인 제한이 있습니다. 올바른 선택은 성능과 디자인의 균형을 맞추는 데 중요합니다.

### 1.1 웹 폰트

서버에서 제공되는 폰트로, 모든 사용자에게 동일한 글꼴을 보여줍니다. 예를 들어, 패션 브랜드, 고급스러운 제품을 홍보하는 웹사이트는 웹 폰트를 활용해 일관된 브랜드 이미지를 유지합니다.

### 1.2 시스템 폰트

사용자의 기기에 설치된 기본 폰트로, 로딩이 필요하지 않아 빠릅니다. 예를 들어, 실시간 데이터 제공 웹사이트(예: 금융 서비스)는 시스템 폰트를 사용해 빠른 로딩을 보장합니다.

## 2. 폰트 포맷 종류

폰트 포맷은 웹 폰트의 성능에 직접적인 영향을 미칩니다. 다양한 포맷의 차이를 이해하면 브라우저 호환성과 성능을 동시에 고려한 최적의 폰트 선택이 가능합니다.

### 2.1 주요 폰트 포맷 설명

- **TTF (TrueType Font)**
  - **특징**: Apple과 Microsoft가 공동 개발한 가장 기본적인 폰트 형식.
  - **장점**: 대부분의 플랫폼에서 지원.
  - **단점**: 압축이 덜 되어 파일 크기가 큼.
- **OTF (OpenType Font)**
  - **특징**: TTF의 확장 버전으로, Adobe와 Microsoft가 공동 개발.
  - **장점**: 고급 타이포그래피 기능과 더 많은 글리프 지원.
  - **단점**: 파일 크기가 TTF보다 크며, WOFF/WOFF2보다 압축 효율이 낮음.
- **WOFF (Web Open Font Format)**
  - **특징**: 웹에 최적화된 포맷으로, TTF와 OTF를 기반으로 함.
  - **장점**: 압축 기능이 있어 파일 크기가 줄어듦.
  - **단점**: WOFF2보다 압축률이 낮음.
- **WOFF2**
  - **특징**: WOFF의 업그레이드 버전으로, 더 높은 압축률 제공.
  - **장점**: 최신 브라우저에서 지원하며, 파일 크기 최소화.
  - **단점**: 구형 브라우저와의 호환성이 낮음.
- **EOT (Embedded OpenType)**
  - **특징**: Microsoft 전용 포맷으로, Internet Explorer를 위해 설계됨.
  - **장점**: IE에서만 사용 가능.
  - **단점**: 다른 브라우저에서 사용 불가능.

### 2.2 폰트 포맷 비교

| 포맷  | 압축률 | 브라우저 호환성        | 주요 사용 사례        |
| ----- | ------ | ---------------------- | --------------------- |
| TTF   | 낮음   | 모든 주요 브라우저     | 문서 편집용 기본 폰트 |
| OTF   | 낮음   | 모든 주요 브라우저     | 고급 타이포그래피     |
| WOFF  | 중간   | 대부분의 최신 브라우저 | 웹 폰트 기본 포맷     |
| WOFF2 | 높음   | 최신 브라우저          | 최적화된 웹 폰트      |
| EOT   | 낮음   | Internet Explorer 전용 | 구형 IE 지원          |

## 3. FOUT와 FOIT

FOUT(Flash of Unstyled Text)와 FOIT(Flash of Invisible Text)는 폰트 로딩 중 발생하는 문제로, 사용자 경험을 크게 저하시킬 수 있습니다. 이 문제를 이해하고 해결하면 사이트의 신뢰성과 만족도를 높일 수 있습니다.

### 3.1 개념 설명

- **FOUT (Flash of Unstyled Text)**
  - 웹 폰트가 로드되기 전, 기본 폰트로 텍스트가 먼저 표시되었다가, 웹 폰트가 로드되면 스타일이 바뀌는 현상.
  - **예**: 웹 폰트를 사용하지만 초기 로딩 시 Arial과 같은 시스템 폰트가 먼저 보이는 경우.
- **FOIT (Flash of Invisible Text)**
  - 웹 폰트가 로드되기 전까지 텍스트가 보이지 않다가, 웹 폰트 로딩 완료 후 한꺼번에 나타나는 현상.
  - **예**: 텍스트가 잠시 동안 비어있는 것처럼 보였다가 갑자기 로드되는 경우.

### 3.2 해결 방법

- **CSS 속성 활용**
  - `font-display: swap;` 속성을 사용해 FOUT를 방지하고, 텍스트가 즉시 표시되도록 설정.
  ```css
  @font-face {
    font-family: 'CustomFont';
    src: url('/fonts/customfont.woff2') format('woff2');
    font-display: swap;
  }
  ```
- **Preload 사용**
  - HTML `<link>` 태그의 `rel="preload"`를 사용해 웹 폰트를 우선 로드.
  ```html
  <link
    rel="preload"
    href="/fonts/customfont.woff2"
    as="font"
    type="font/woff2"
    crossorigin="anonymous"
  />
  ```
